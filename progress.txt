## Codebase Patterns
- Entity classes extend abstract `Entity` in `src/core/Entity.ts` — implement `getBoundingBox`, `getGripPoints`, `draw`, `hitTest`, `move`, `moveGripPoint`, `clone`
- Tool classes extend `BaseTool` in `src/core/tools/Tool.ts` — implement `name`, `onMouseDown/Move/Up`, `onKeyDown`, `drawOverlay`
- Each tool has a static `parseCommandLine(args)` method for command-line creation
- Tools register via `ToolManager.register()` in `CAD.ts` setupTools()
- Commands handled in `CAD.ts` `executeCommand()` switch block
- `drawOverlay` receives `{ worldToScreen }` — NOT the full Viewport; compute screen distances via `Math.hypot` on screen coords
- Exports go in both `src/core/tools/index.ts` and `src/index.ts`
- Demo at `demo/index.html` needs toolbar click handler + keyboard shortcut for each new tool
- `EntityManager` handles all entity CRUD, selection, and hit testing
- Grip point types: `'endpoint' | 'midpoint' | 'center' | 'quadrant'`
- Entity types: `'line' | 'circle' | 'arc'`
- Build: `tsc && vite build` — typecheck first, then bundle
- Run `npx tsc --noEmit` for quick typecheck without building
- DXF operations: `DXFParser.parse(content)` → `{ entities, layers, version }`; `HowdzCAD.loadDXF(content)` adds parsed entities + auto zoom-extents
- DXF export: `DXFWriter.export(entities, layers?)` → DXF string; `HowdzCAD.saveDXF()` exports current entities; demo triggers download via Blob+URL.createObjectURL
- File I/O (open/save) lives in demo layer via hidden `<input type="file">` (open) and Blob+`<a>` click (save) — core library has no DOM file API
