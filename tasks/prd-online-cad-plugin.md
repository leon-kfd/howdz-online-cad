# PRD: 纯前端在线CAD插件 (howdz-online-cad)

## Introduction

开发一个基于Canvas 2D的纯前端在线CAD插件，以可嵌入组件库的形式提供，支持集成到任意前端项目中。插件采用经典AutoCAD风格界面（顶部菜单+左侧工具栏+底部命令行），提供基础二维图形绘制与编辑功能，支持常见CAD命令操作，并能导入导出多版本DXF文件。

**目标定位：** 轻量级、易集成、功能实用的前端CAD解决方案，适用于需要基础CAD绘图能力的Web应用。

## Goals

- 提供可嵌入的前端CAD组件，支持npm安装或CDN引入
- 实现Canvas 2D渲染引擎，支持基础二维图形绘制
- 支持直线、圆形、圆弧三种基础图形元素
- 提供复制、移动、倒圆角、倒角、延伸、清除等编辑功能
- 实现类AutoCAD命令行交互（底部命令输入框+快捷键触发）
- 支持打开（导入）和保存（导出）DXF文件，兼容多版本格式
- 采用经典AutoCAD风格布局：顶部菜单栏、左侧工具栏、绘图区域、底部命令行

## User Stories

### US-001: CAD画布基础渲染
**Description:** 作为用户，我需要一个可渲染二维图形的画布区域，以便进行绘图操作。

**Acceptance Criteria:**
- [ ] 实现Canvas 2D渲染画布，支持无限滚动和平移（鼠标中键拖拽）
- [ ] 支持缩放（鼠标滚轮），缩放范围0.01x到100x
- [ ] 显示网格背景（可开关），网格随缩放自适应密度
- [ ] 显示坐标原点标记和十字光标
- [ ] 渲染帧率≥30fps（1000个图元场景下）
- [ ] 验证在浏览器中正常显示画布和网格

### US-002: 直线绘制
**Description:** 作为用户，我想要绘制直线段，以便构建基础图形。

**Acceptance Criteria:**
- [ ] 支持鼠标点击两点绘制直线
- [ ] 支持命令行输入坐标绘制：`LINE 0,0 100,100`
- [ ] 支持连续绘制模式（自动以前一个终点为起点）
- [ ] 支持正交模式（Ortho，按F8切换），限制为水平/垂直方向
- [ ] 直线可被选中，选中时显示夹点（端点和中点）
- [ ] 验证在浏览器中可正常绘制和选中直线

### US-003: 圆形绘制
**Description:** 作为用户，我想要绘制圆形，以便添加圆形元素。

**Acceptance Criteria:**
- [ ] 支持圆心+半径模式：点击圆心，拖拽确定半径
- [ ] 支持两点定圆：点击直径两端点
- [ ] 支持命令行输入：`CIRCLE 50,50 25`（圆心x,y 半径）
- [ ] 圆形可被选中，选中时显示圆心和四个象限夹点
- [ ] 验证在浏览器中可正常绘制圆形

### US-004: 圆弧绘制
**Description:** 作为用户，我想要绘制圆弧，以便创建弧形线条。

**Acceptance Criteria:**
- [ ] 支持三点画弧：起点、经过点、终点
- [ ] 支持圆心+起点+角度模式
- [ ] 支持命令行输入：`ARC 0,0 50,0 25,25`（三点坐标）
- [ ] 圆弧可被选中，选中时显示端点和中点夹点
- [ ] 验证在浏览器中可正常绘制圆弧

### US-005: 图形选择与夹点编辑
**Description:** 作为用户，我想要选择图形并通过拖拽夹点修改形状。

**Acceptance Criteria:**
- [ ] 单击选择单个图元，选中高亮显示（蓝色边框）
- [ ] 框选（从左到右拖拽）选择区域内所有图元
- [ ] 交叉选择（从右到左拖拽）选择与框相交的所有图元
- [ ] 按住Shift点击可从选择集中移除图元
- [ ] 拖拽夹点可实时修改图形形状
- [ ] 选中图元后显示属性面板（坐标、长度等）
- [ ] 验证在浏览器中可正常选择和编辑图形

### US-006: 复制功能
**Description:** 作为用户，我想要复制选中的图形到指定位置。

**Acceptance Criteria:**
- [ ] 选择图元后，点击复制命令或输入`COPY`
- [ ] 指定基点和目标点（或输入位移偏移量）
- [ ] 支持多重复制（连续指定多个目标点，按Enter结束）
- [ ] 复制后原图保留，新图出现在目标位置
- [ ] 快捷键`Ctrl+C`可复制选中图元到剪贴板
- [ ] 验证在浏览器中可正常复制图形

### US-007: 移动功能
**Description:** 作为用户，我想要移动选中的图形到指定位置。

**Acceptance Criteria:**
- [ ] 选择图元后，点击移动命令或输入`MOVE`
- [ ] 指定基点和目标点（或输入位移偏移量）
- [ ] 移动后原位置图形消失，新位置出现
- [ ] 快捷键`Ctrl+V`可从剪贴板粘贴
- [ ] 验证在浏览器中可正常移动图形

### US-008: 倒圆角（Fillet）
**Description:** 作为用户，我想要对两条线段进行倒圆角处理。

**Acceptance Criteria:**
- [ ] 输入`FILLET`命令或点击工具栏按钮
- [ ] 设置圆角半径（默认10，可通过命令行修改）
- [ ] 依次点击两条线段，自动生成圆弧连接
- [ ] 支持直线-直线、直线-圆弧、圆弧-圆弧的组合
- [ ] 如果两线不相交，自动延伸后倒圆角
- [ ] 验证在浏览器中可正常执行倒圆角

### US-009: 倒角（Chamfer）
**Description:** 作为用户，我想要对两条线段进行倒角处理。

**Acceptance Criteria:**
- [ ] 输入`CHAMFER`命令或点击工具栏按钮
- [ ] 设置倒角距离（默认10，可分别设置两个距离）
- [ ] 依次点击两条线段，自动生成倒角线
- [ ] 支持等距倒角和不等距倒角
- [ ] 验证在浏览器中可正常执行倒角

### US-010: 延伸（Extend）
**Description:** 作为用户，我想要将线段延伸到指定边界。

**Acceptance Criteria:**
- [ ] 输入`EXTEND`命令或点击工具栏按钮
- [ ] 先选择边界线，再选择要延伸的线段
- [ ] 线段自动延伸至与边界线相交
- [ ] 支持延伸直线到圆弧、圆弧到直线
- [ ] 验证在浏览器中可正常延伸线段

### US-011: 清除（Erase）
**Description:** 作为用户，我想要删除选中的图形。

**Acceptance Criteria:**
- [ ] 选择图元后，按`Delete`键或输入`ERASE`命令删除
- [ ] 支持清除全部：`ERASE ALL`或工具栏按钮
- [ ] 删除前显示确认对话框（可选配置）

### US-012: 命令行交互
**Description:** 作为用户，我想要通过底部命令行输入CAD命令来操作图形。

**Acceptance Criteria:**
- [ ] 底部显示命令输入框，显示当前命令提示符（如`命令:`）

- [ ] 命令行支持输入坐标值（绝对坐标`50,100`和相对坐标`@50,100`）
- [ ] 验证在浏览器中可正常输入命令并执行

## Functional Requirements

### 核心绘图功能
- **FR-1:** 实现Canvas 2D渲染引擎，支持画布平移（鼠标中键拖拽）和缩放（滚轮）
- **FR-2:** 实现直线绘制，支持鼠标点击绘制和命令行坐标输入
- **FR-3:** 实现圆形绘制，支持圆心+半径模式和两点定圆模式
- **FR-4:** 实现圆弧绘制，支持三点画弧和圆心+起点+角度模式
- **FR-5:** 实现图形选择功能，支持单选、框选、交叉选择
- **FR-6:** 实现夹点编辑，拖拽夹点可修改图形形状

### 编辑功能
- **FR-7:** 实现复制功能，支持基点+目标点复制和多重复制
- **FR-8:** 实现移动功能，支持基点+目标点移动
- **FR-9:** 实现倒圆角功能，支持设置圆角半径，处理直线-直线、直线-圆弧组合
- **FR-10:** 实现倒角功能，支持等距和不等距倒角
- **FR-11:** 实现延伸功能，支持将线段延伸至指定边界
- **FR-12:** 实现删除功能，支持Delete键和ERASE命令删除
- **FR-13:** 实现撤销/重做功能，支持Ctrl+Z/Ctrl+Y

### 命令系统
- **FR-14:** 实现底部命令行输入框，支持输入命令名和参数
- **FR-15:** 实现命令缩写映射（L→LINE, C→CIRCLE, A→ARC等）
- **FR-16:** 实现坐标输入解析，支持绝对坐标和相对坐标（@前缀）
- **FR-17:** 实现命令历史记录，支持上下箭头翻阅
- **FR-18:** 实现快捷键绑定系统，支持单键和组合键

### DXF文件处理
- **FR-19:** 实现DXF解析器，支持读取LINE、CIRCLE、ARC实体
- **FR-20:** 实现DXF版本检测，支持R12至R32多版本解析
- **FR-21:** 实现DXF图层解析，保留图层名称、颜色、可见性
- **FR-22:** 实现DXF生成器，将画布图形导出为DXF格式
- **FR-23:** 实现文件选择对话框集成（`<input type="file">`）
- **FR-24:** 实现文件下载功能（Blob + URL.createObjectURL）

### 界面功能
- **FR-25:** 实现顶部菜单栏，包含文件、编辑、视图、绘图、修改、帮助菜单
- **FR-26:** 实现左侧工具栏，包含所有绘图和编辑工具按钮
- **FR-27:** 实现底部命令行区域，支持展开/折叠
- **FR-28:** 实现右侧属性面板，显示选中图元属性
- **FR-29:** 实现状态栏，显示鼠标坐标、图层、正交/捕捉状态
- **FR-30:** 实现网格显示，可开关，随缩放自适应

## Non-Goals (Out of Scope)

以下功能在本版本中**不实现**：

- ❌ 三维图形绘制和渲染
- ❌ 样条曲线（Spline）、椭圆（Ellipse）、多段线（Polyline）等复杂图形
- ❌ 图案填充（Hatch）和渐变填充
- ❌ 尺寸标注和文字注释
- ❌ 块（Block）定义和插入
- ❌ 外部引用（Xref）
- ❌ 打印输出功能
- ❌ 多文档/多标签页同时编辑
- ❌ 协同编辑/实时同步
- ❌ 自定义菜单和工具栏配置
- ❌ 脚本/自动化编程接口（LISP等）
- ❌ DWG文件格式支持（仅支持DXF）
- ❌ 移动端适配（仅支持桌面浏览器）

### 图形数据模型
```javascript
// 基础图元接口
interface Entity {
  id: string;                  // 唯一ID
  type: 'line' | 'circle' | 'arc';
  layer: string;               // 所属图层
  color: string;               // 颜色（默认随图层）
  properties: Record<string, any>;  // 图元属性
}

// 直线
interface Line extends Entity {
  type: 'line';
  properties: {
    startX: number;
    startY: number;
    endX: number;
    endY: number;
  };
}

// 圆形
interface Circle extends Entity {
  type: 'circle';
  properties: {
    centerX: number;
    centerY: number;
    radius: number;
  };
}

// 圆弧
interface Arc extends Entity {
  type: 'arc';
  properties: {
    centerX: number;
    centerY: number;
    radius: number;
    startAngle: number;        // 弧度
    endAngle: number;          // 弧度
  };
}
```

## Technical Considerations

### 技术栈选择
- **渲染引擎：** Canvas 2D API（原生，无第三方依赖）
- **DXF解析：** 使用`dxf-parser`库或自研解析器
- **构建工具：** Vite/Rollup（ESM + UMD双格式输出）
- **包管理：** npm发布，支持CDN引入（unpkg/jsdelivr）
- **TypeScript：** 使用TypeScript开发，提供完整类型定义

### 性能要求
- **渲染性能：** 1000个图元场景下保持≥30fps
- **内存占用：** 10000个图元场景下内存<100MB
- **文件导入：** 1MB DXF文件导入时间<3秒
- **初始化时间：** 组件挂载到首次渲染<500ms

### 兼容性要求
- **浏览器：** Chrome 90+、Firefox 90+、Safari 15+、Edge 90+
- **屏幕分辨率：** 最低1366×768，推荐1920×1080
- **输入设备：** 鼠标（必须）、键盘（必须）

## Open Questions

1. **DXF解析库选择：** 是使用开源库（如`dxf-parser`）还是完全自研？开源库可能不支持所有版本，自研工作量大但可控性强。

2. **图层颜色映射：** DXF使用ACI（AutoCAD Color Index）颜色系统，是否需要支持真彩色（RGB）还是仅支持256色ACI索引？

3. **单位系统：** DXF文件包含单位信息（毫米、厘米、米等），是否需要在界面上显示当前单位并支持单位切换？

4. **视图状态持久化：** 是否需要保存用户的视图状态（缩放比例、平移位置）到localStorage？

5. **错误处理策略：** DXF文件损坏或格式不兼容时，应该显示错误对话框还是静默忽略不支持的实体？

6. **键盘冲突处理：** 当用户在命令行输入框输入时，是否应该禁用所有快捷键以避免冲突？

7. **性能优化：** 对于超大DXF文件（>10MB），是否需要实现分块加载或LOD（Level of Detail）渲染？
